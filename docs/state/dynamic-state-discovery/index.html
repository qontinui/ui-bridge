Dynamic State Discovery

Dynamic State Discovery automatically learns and tracks application states without explicit configuration. The system observes UI changes and builds a state model over time.

Overview​

Dynamic state discovery:

  • Automatically detects distinct UI states
  • Learns transitions between states
  • Identifies state indicators
  • Builds a navigable state graph

Enabling Discovery​

Basic Setup​

import { StateDiscovery } from '@anthropic/ui-bridge';

const discovery = new StateDiscovery(registry, {
  enabled: true,
  learningMode: 'active',
});

// Start discovery
await discovery.start();

With React Provider​

import { UIBridgeProvider } from '@anthropic/ui-bridge/react';

function App() {
  return (
    <UIBridgeProvider
      stateDiscovery={{
        enabled: true,
        learningMode: 'passive',
      }}
    >
      {children}
    </UIBridgeProvider>
  );
}

Learning Modes​

Passive Learning​

Learns by observing user interactions:

const discovery = new StateDiscovery(registry, {
  learningMode: 'passive',
  // Only observe, don't take actions
});

Active Learning​

Actively explores the UI to discover states:

const discovery = new StateDiscovery(registry, {
  learningMode: 'active',
  explorationStrategy: 'breadth-first',
  maxExplorationDepth: 5,
  excludePatterns: ['/admin/*', '/api/*'],
});

State Signatures​

States are identified by signatures:

interface StateSignature {
  url: string;
  urlPattern?: RegExp;
  domFingerprint: string;
  visibleElements: string[];
  forms: FormSignature[];
  hash: string;
}

Custom Signature Components​

discovery.configure({
  signatureComponents: {
    url: true,
    title: true,
    mainContent: '.main-content',
    forms: true,
    customElements: ['#app-state', '.page-id'],
  },
});

Discovered States​

List States​

const states = discovery.getDiscoveredStates();

states.forEach(state => {
  console.log(`${state.id}: ${state.description}`);
  console.log(`  URL: ${state.signature.url}`);
  console.log(`  Elements: ${state.signature.visibleElements.length}`);
});

State Details​

const state = discovery.getState('state-abc123');

console.log(state);
// {
//   id: 'state-abc123',
//   description: 'Login page',
//   signature: {...},
//   discoveredAt: '2024-01-15T10:30:00Z',
//   visitCount: 42,
//   transitions: ['state-def456', 'state-ghi789'],
// }

Discovered Transitions​

List Transitions​

const transitions = discovery.getDiscoveredTransitions();

transitions.forEach(t => {
  console.log(`${t.from} -> ${t.to}`);
  console.log(`  Action: ${t.action.type} on ${t.action.target}`);
  console.log(`  Success rate: ${t.successRate}%`);
});

Transition Details​

const transition = discovery.getTransition('state-abc123', 'state-def456');

console.log(transition);
// {
//   from: 'state-abc123',
//   to: 'state-def456',
//   action: { type: 'click', target: '#login-btn' },
//   successRate: 95,
//   avgDuration: 1200,
//   lastUsed: '2024-01-15T10:35:00Z',
// }

Export and Import​

Export Discovered Model​

const model = discovery.export();

// Save to file
await fs.writeFile('state-model.json', JSON.stringify(model, null, 2));

Import Model​

const model = JSON.parse(await fs.readFile('state-model.json'));
discovery.import(model);

Convert to State Machine​

const stateMachine = discovery.toStateMachine();

// Now use with StateMachine API
await stateMachine.navigateTo('checkout');

Discovery Events​

discovery.on('stateDiscovered', (state) => {
  console.log(`New state discovered: ${state.id}`);
});

discovery.on('transitionDiscovered', (transition) => {
  console.log(`New transition: ${transition.from} -> ${transition.to}`);
});

discovery.on('stateUpdated', (state) => {
  console.log(`State updated: ${state.id}`);
});

State Merging​

Similar states can be merged:

// Configure similarity threshold
discovery.configure({
  mergeThreshold: 0.85,  // 85% similarity
  autoMerge: true,
});

// Manual merge
discovery.mergeStates('state-abc123', 'state-def456');

State Naming​

Auto-Generated Names​

States get automatic names based on URL and content:

// Auto-generated: "login-page", "dashboard-main", "settings-profile"

Custom Names​

discovery.renameState('state-abc123', 'user-login');

Naming Rules​

discovery.configure({
  namingRules: [
    { pattern: /\/login/, name: 'login' },
    { pattern: /\/dashboard/, name: 'dashboard' },
    { selector: '#checkout-form', name: 'checkout' },
  ],
});

Configuration​

const discovery = new StateDiscovery(registry, {
  enabled: true,
  learningMode: 'passive',

  // Signature configuration
  signatureComponents: {
    url: true,
    title: true,
    forms: true,
  },

  // Exploration settings (active mode)
  exploration: {
    maxDepth: 5,
    maxStates: 100,
    timeout: 300000,  // 5 minutes
  },

  // Merging
  mergeThreshold: 0.85,
  autoMerge: false,

  // Persistence
  persistTo: 'localStorage',
  persistKey: 'ui-bridge-states',
});

Best Practices​

  1. Start Passive - Begin with passive learning to avoid unintended actions
  2. Set Exclusions - Exclude admin, logout, and destructive pages from active exploration
  3. Review Merges - Manually review auto-merged states for accuracy
  4. Export Regularly - Save discovered models for reuse
  5. Name Important States - Give meaningful names to frequently used states
  6. Validate Transitions - Test discovered transitions before relying on them
