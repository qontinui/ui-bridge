State Machine Integration

UI Bridge integrates with the multistate library to provide explicit state management for UI automation. This enables model-based testing and intelligent navigation.

Overview​

State machine integration provides:

  • Explicit modeling of UI states
  • Defined transitions between states
  • Automatic state detection
  • Path planning through state graphs

Basic Setup​

Define States​

import { StateMachine } from '@anthropic/ui-bridge';

const machine = new StateMachine({
  states: {
    'logged-out': {
      description: 'User is not authenticated',
      indicators: [
        { type: 'element-exists', selector: '#login-form' },
        { type: 'url-matches', pattern: /\/(login|signin)/ },
      ],
    },
    'logged-in': {
      description: 'User is authenticated',
      indicators: [
        { type: 'element-exists', selector: '#user-menu' },
        { type: 'element-not-exists', selector: '#login-form' },
      ],
    },
    'dashboard': {
      description: 'Main dashboard view',
      indicators: [
        { type: 'url-matches', pattern: /\/dashboard/ },
        { type: 'element-exists', selector: '.dashboard-grid' },
      ],
    },
  },
});

Define Transitions​

machine.addTransitions({
  'logged-out -> logged-in': {
    description: 'User logs in',
    actions: [
      { action: 'type', target: '#email', value: '{{email}}' },
      { action: 'type', target: '#password', value: '{{password}}' },
      { action: 'click', target: '#login-btn' },
    ],
    guards: [
      { type: 'element-enabled', selector: '#login-btn' },
    ],
  },
  'logged-in -> logged-out': {
    description: 'User logs out',
    actions: [
      { action: 'click', target: '#user-menu' },
      { action: 'click', target: '#logout-btn' },
    ],
  },
  'logged-in -> dashboard': {
    description: 'Navigate to dashboard',
    actions: [
      { action: 'click', target: '#dashboard-link' },
    ],
  },
});

State Detection​

Get Current State​

const currentState = await machine.getCurrentState();
console.log(currentState); // 'logged-out'

Watch State Changes​

machine.onStateChange((from, to) => {
  console.log(`State changed: ${from} -> ${to}`);
});

Validate State​

const isValid = await machine.validateState('logged-in');
console.log(isValid); // true if all indicators match

State Navigation​

Navigate to State​

// Automatically finds and executes path to target state
await machine.navigateTo('dashboard', {
  credentials: {
    email: 'user@example.com',
    password: 'secret',
  },
});

Get Path​

const path = machine.findPath('logged-out', 'dashboard');
console.log(path);
// ['logged-out -> logged-in', 'logged-in -> dashboard']

Execute Transition​

await machine.executeTransition('logged-out -> logged-in', {
  email: 'user@example.com',
  password: 'secret',
});

Integration with Registry​

Auto-Register States​

import { UIBridgeProvider, useStateMachine } from '@anthropic/ui-bridge/react';

function App() {
  return (
    <UIBridgeProvider
      stateMachine={machine}
      autoDetectState={true}
    >
      <Content />
    </UIBridgeProvider>
  );
}

Use in Components​

function Dashboard() {
  const { currentState, navigateTo } = useStateMachine();

  return (
    <div>
      <p>Current state: {currentState}</p>
      <button onClick={() => navigateTo('settings')}>
        Go to Settings
      </button>
    </div>
  );
}

State Indicators​

Element-Based​

{
  type: 'element-exists',
  selector: '#user-menu',
}

{
  type: 'element-not-exists',
  selector: '#login-form',
}

{
  type: 'element-visible',
  selector: '.dashboard',
}

{
  type: 'element-has-text',
  selector: '#status',
  text: 'Connected',
}

URL-Based​

{
  type: 'url-matches',
  pattern: /\/dashboard/,
}

{
  type: 'url-equals',
  url: '/settings',
}

Custom​

{
  type: 'custom',
  check: async () => {
    const response = await fetch('/api/auth/status');
    const data = await response.json();
    return data.authenticated;
  },
}

Transition Guards​

Prevent transitions under certain conditions:

machine.addTransition('logged-in -> admin', {
  actions: [...],
  guards: [
    {
      type: 'custom',
      check: async () => {
        const user = await getCurrentUser();
        return user.role === 'admin';
      },
      message: 'User must be an admin',
    },
  ],
});

State Persistence​

Save and restore state machine state:

// Save
const snapshot = machine.serialize();
localStorage.setItem('state-machine', JSON.stringify(snapshot));

// Restore
const saved = JSON.parse(localStorage.getItem('state-machine'));
machine.restore(saved);

Visualization​

Generate state diagram:

const diagram = machine.toMermaid();
console.log(diagram);
// stateDiagram-v2
//   [*] --> logged_out
//   logged_out --> logged_in: login
//   logged_in --> dashboard: navigate
//   logged_in --> logged_out: logout

Best Practices​

  1. Model Real States - States should reflect actual UI states
  2. Clear Indicators - Use reliable indicators for state detection
  3. Guard Transitions - Prevent invalid transitions with guards
  4. Parameterize Actions - Use variables for credentials and data
  5. Test Paths - Verify all paths are navigable
