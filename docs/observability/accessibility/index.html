Accessibility

UI Bridge includes accessibility features for WCAG validation, ARIA attribute extraction, and creating accessible automation.

Overview​

Accessibility features provide:

  • WCAG guideline validation
  • ARIA attribute discovery
  • Accessibility tree extraction
  • Screen reader compatibility
  • Inclusive automation patterns

WCAG Validation​

Run Validation​

import { validateAccessibility } from '@anthropic/ui-bridge';

const results = await validateAccessibility({
  level: 'AA',  // 'A' | 'AA' | 'AAA'
  scope: document.body,
});

console.log(results);
// {
//   passed: 45,
//   failed: 3,
//   warnings: 7,
//   violations: [...],
// }

Validation Results​

interface ValidationResult {
  passed: number;
  failed: number;
  warnings: number;
  violations: Violation[];
}

interface Violation {
  rule: string;
  level: 'A' | 'AA' | 'AAA';
  element: Element;
  description: string;
  howToFix: string;
  wcagCriteria: string;
}

Example Violations​

results.violations.forEach(v => {
  console.log(`[${v.level}] ${v.rule}`);
  console.log(`  Element: ${v.element.tagName}`);
  console.log(`  Issue: ${v.description}`);
  console.log(`  Fix: ${v.howToFix}`);
  console.log(`  WCAG: ${v.wcagCriteria}`);
});

// Output:
// [AA] color-contrast
//   Element: BUTTON
//   Issue: Text has insufficient color contrast
//   Fix: Ensure contrast ratio is at least 4.5:1
//   WCAG: 1.4.3

ARIA Extraction​

Get ARIA Attributes​

import { getAriaAttributes } from '@anthropic/ui-bridge';

const aria = getAriaAttributes(element);

console.log(aria);
// {
//   role: 'button',
//   label: 'Submit form',
//   describedBy: 'help-text',
//   expanded: false,
//   pressed: false,
//   disabled: false,
// }

Accessibility Tree​

import { getAccessibilityTree } from '@anthropic/ui-bridge';

const tree = await getAccessibilityTree();

console.log(tree);
// {
//   role: 'document',
//   name: 'Login Page',
//   children: [
//     {
//       role: 'main',
//       children: [
//         { role: 'form', name: 'Login Form', children: [...] },
//       ],
//     },
//   ],
// }

Accessible Element Discovery​

Find by Accessible Name​

const elements = await registry.findByAccessibleName('Submit');
// Finds elements with aria-label="Submit" or visible text "Submit"

Find by Role​

const buttons = await registry.findByRole('button');
const inputs = await registry.findByRole('textbox');
const links = await registry.findByRole('link');

Find by Landmark​

const main = await registry.findByLandmark('main');
const nav = await registry.findByLandmark('navigation');
const footer = await registry.findByLandmark('contentinfo');

Accessible Automation​

Use Accessible Names​

// Prefer accessible names over selectors
await registry.executeByName('Submit form', 'click');
await registry.executeByName('Email address', 'type', 'user@example.com');

Wait for Accessible State​

await registry.waitForAccessibleState('Submit', {
  role: 'button',
  disabled: false,
});

Keyboard Navigation​

import { pressKey, tabTo } from '@anthropic/ui-bridge';

// Navigate using keyboard
await tabTo('Email input');
await pressKey('Tab');
await pressKey('Enter');

// Use keyboard shortcuts
await pressKey('Escape');  // Close modal
await pressKey('Space');   // Activate button

Screen Reader Support​

Announce Actions​

import { announce } from '@anthropic/ui-bridge';

// Announce to screen readers
announce('Form submitted successfully', 'polite');
announce('Error: Invalid email address', 'assertive');

Get Live Regions​

const liveRegions = await registry.findLiveRegions();

liveRegions.forEach(region => {
  console.log(`Live region: ${region.role} - ${region.ariaLive}`);
});

React Integration​

AccessibilityProvider​

import { UIBridgeProvider } from '@anthropic/ui-bridge/react';

function App() {
  return (
    <UIBridgeProvider
      accessibility={{
        validation: true,
        level: 'AA',
        announceActions: true,
      }}
    >
      {children}
    </UIBridgeProvider>
  );
}

useAccessibility Hook​

function MyComponent() {
  const {
    validate,
    violations,
    getAriaProps
  } = useAccessibility();

  return (
    <button {...getAriaProps('Submit the form')}>
      Submit
    </button>
  );
}

Configuration​

import { configureAccessibility } from '@anthropic/ui-bridge';

configureAccessibility({
  // Validation
  validation: {
    enabled: true,
    level: 'AA',
    runOnChange: true,
    excludeSelectors: ['.third-party-widget'],
  },

  // Discovery
  discovery: {
    preferAccessibleNames: true,
    fallbackToText: true,
  },

  // Screen reader
  announcements: {
    enabled: true,
    defaultPoliteness: 'polite',
  },

  // Keyboard
  keyboard: {
    enableTabNavigation: true,
    trapFocusInModals: true,
  },
});

Reports​

Generate Report​

import { generateAccessibilityReport } from '@anthropic/ui-bridge';

const report = await generateAccessibilityReport({
  format: 'html',  // 'html' | 'json' | 'csv'
  includeScreenshots: true,
});

await fs.writeFile('accessibility-report.html', report);

CI Integration​

const results = await validateAccessibility({ level: 'AA' });

if (results.failed > 0) {
  console.error(`Accessibility check failed: ${results.failed} violations`);
  process.exit(1);
}

Best Practices​

  1. Use Accessible Names - Prefer aria-label and role over CSS selectors
  2. Validate Regularly - Run WCAG validation in CI/CD
  3. Test with Keyboard - Ensure all actions work with keyboard
  4. Support Screen Readers - Add meaningful announcements
  5. Include in Automation - Make accessibility part of your test suite
  6. Fix Violations - Address violations before they reach production
