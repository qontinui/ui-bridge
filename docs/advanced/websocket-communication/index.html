WebSocket Communication

UI Bridge supports WebSocket communication for real-time, bidirectional interaction with the UI. This provides lower latency than HTTP polling and enables push-based updates.

Overview​

WebSocket communication provides:

  • Real-time action execution
  • Live state updates
  • Event streaming
  • Automatic reconnection
  • HTTP fallback support

Enabling WebSocket​

Server-Side​

import { createUIBridgeServer } from '@anthropic/ui-bridge-server';

const server = createUIBridgeServer({
  transport: 'websocket',
  port: 9876,
  path: '/ws',
});

server.start();

Client-Side​

import { UIBridgeClient } from '@anthropic/ui-bridge';

const client = new UIBridgeClient({
  transport: 'websocket',
  url: 'ws://localhost:9876/ws',
});

await client.connect();

Message Protocol​

Message Format​

interface Message {
  type: string;
  id: string;
  payload: any;
  timestamp: number;
}

Request/Response​

// Request
{
  "type": "execute",
  "id": "msg-123",
  "payload": {
    "action": "click",
    "elementId": "submit-btn"
  },
  "timestamp": 1705312200000
}

// Response
{
  "type": "execute.result",
  "id": "msg-123",
  "payload": {
    "success": true,
    "duration": 150
  },
  "timestamp": 1705312200150
}

Event Messages​

// State change event
{
  "type": "event.stateChange",
  "id": "evt-456",
  "payload": {
    "from": "logged-out",
    "to": "logged-in"
  },
  "timestamp": 1705312200000
}

Client Usage​

Execute Actions​

const result = await client.execute('submit-btn', 'click');

Subscribe to Events​

client.on('stateChange', (event) => {
  console.log(`State: ${event.from} -> ${event.to}`);
});

client.on('elementAdded', (event) => {
  console.log(`New element: ${event.elementId}`);
});

client.on('elementRemoved', (event) => {
  console.log(`Removed: ${event.elementId}`);
});

Stream Updates​

// Subscribe to registry updates
const unsubscribe = client.subscribeToRegistry((snapshot) => {
  console.log(`Elements: ${snapshot.elements.length}`);
});

// Later: unsubscribe()

Connection Management​

Auto-Reconnection​

const client = new UIBridgeClient({
  transport: 'websocket',
  url: 'ws://localhost:9876/ws',
  reconnect: {
    enabled: true,
    maxAttempts: 10,
    delay: 1000,
    backoff: 'exponential',
  },
});

client.on('disconnected', () => {
  console.log('Connection lost, reconnecting...');
});

client.on('reconnected', () => {
  console.log('Connection restored');
});

Connection Status​

console.log(client.status);
// 'connecting' | 'connected' | 'disconnected' | 'reconnecting'

client.on('statusChange', (status) => {
  console.log(`Connection status: ${status}`);
});

Manual Disconnect​

await client.disconnect();

HTTP Fallback​

When WebSocket is unavailable:

const client = new UIBridgeClient({
  transport: 'websocket',
  url: 'ws://localhost:9876/ws',
  fallback: {
    enabled: true,
    transport: 'http',
    url: 'http://localhost:9876/api',
    pollInterval: 500,
  },
});

Server Events​

Push Updates to Clients​

// Send to all clients
server.broadcast({
  type: 'notification',
  payload: { message: 'Server restarting in 60 seconds' },
});

// Send to specific client
server.send(clientId, {
  type: 'result',
  payload: { success: true },
});

Handle Client Messages​

server.on('message', (clientId, message) => {
  console.log(`Message from ${clientId}:`, message);
});

server.on('execute', async (clientId, { elementId, action }) => {
  const result = await registry.executeAction(elementId, action);
  server.send(clientId, { type: 'execute.result', payload: result });
});

Authentication​

Token-Based​

const client = new UIBridgeClient({
  transport: 'websocket',
  url: 'ws://localhost:9876/ws',
  auth: {
    type: 'token',
    token: 'your-auth-token',
  },
});

Custom Headers​

const client = new UIBridgeClient({
  transport: 'websocket',
  url: 'ws://localhost:9876/ws',
  headers: {
    'Authorization': 'Bearer token',
    'X-Client-ID': 'client-123',
  },
});

Multiplexing​

Support multiple channels over one connection:

// Subscribe to specific channels
const registryChannel = client.channel('registry');
const eventsChannel = client.channel('events');

registryChannel.on('update', (data) => {
  // Handle registry updates
});

eventsChannel.on('action', (data) => {
  // Handle action events
});

Configuration​

Client Options​

const client = new UIBridgeClient({
  transport: 'websocket',
  url: 'ws://localhost:9876/ws',

  // Connection
  timeout: 10000,
  pingInterval: 30000,

  // Reconnection
  reconnect: {
    enabled: true,
    maxAttempts: 10,
    delay: 1000,
    backoff: 'exponential',
  },

  // Fallback
  fallback: {
    enabled: true,
    transport: 'http',
    url: 'http://localhost:9876/api',
  },

  // Buffering
  messageBuffer: {
    enabled: true,
    maxSize: 100,
    flushOnReconnect: true,
  },
});

Server Options​

const server = createUIBridgeServer({
  transport: 'websocket',
  port: 9876,
  path: '/ws',

  // Connection limits
  maxConnections: 100,
  idleTimeout: 60000,

  // Message handling
  maxMessageSize: 1024 * 1024,  // 1MB

  // Security
  cors: {
    origin: '*',
  },
});

Best Practices​

  1. Use WebSocket for Real-Time - Prefer WebSocket when low latency matters
  2. Enable Fallback - Always have HTTP fallback for compatibility
  3. Handle Disconnections - Implement reconnection logic
  4. Buffer Messages - Buffer during disconnection
  5. Authenticate Connections - Secure WebSocket endpoints
  6. Monitor Connection Health - Track ping/pong for connection status
