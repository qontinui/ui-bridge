Cross-Framework Support

UI Bridge supports multiple frontend frameworks beyond React, including Vue, Angular, Svelte, and vanilla JavaScript.

Overview​

Cross-framework support includes:

  • Framework-specific adapters
  • Automatic element discovery
  • Consistent API across frameworks
  • SWC/Babel plugins for each framework

Vue Integration​

Installation​

npm install @anthropic/ui-bridge-vue

Setup​

// main.ts
import { createApp } from 'vue';
import { UIBridgePlugin } from '@anthropic/ui-bridge-vue';
import App from './App.vue';

const app = createApp(App);
app.use(UIBridgePlugin, {
  autoRegister: true,
  idAttribute: 'data-ui-id',
});
app.mount('#app');

Vue Components​

<template>
  <button v-ui-bridge="'submit-btn'" @click="submit">
    Submit
  </button>
</template>

<script setup>
import { useUIBridge } from '@anthropic/ui-bridge-vue';

const { registry, executeAction } = useUIBridge();

async function submit() {
  await executeAction('submit-btn', 'click');
}
</script>

Vue Directive​

<template>
  <!-- Automatic ID generation -->
  <button v-ui-bridge @click="submit">Submit</button>

  <!-- Explicit ID -->
  <button v-ui-bridge="'custom-id'" @click="submit">Submit</button>

  <!-- With options -->
  <button v-ui-bridge="{ id: 'submit', aliases: ['send', 'confirm'] }">
    Submit
  </button>
</template>

Angular Integration​

Installation​

npm install @anthropic/ui-bridge-angular

Setup​

// app.module.ts
import { NgModule } from '@angular/core';
import { UIBridgeModule } from '@anthropic/ui-bridge-angular';

@NgModule({
  imports: [
    UIBridgeModule.forRoot({
      autoRegister: true,
    }),
  ],
})
export class AppModule {}

Angular Components​

// component.ts
import { Component } from '@angular/core';
import { UIBridgeService } from '@anthropic/ui-bridge-angular';

@Component({
  selector: 'app-login',
  template: `
    <button uiBridge="submit-btn" (click)="submit()">
      Submit
    </button>
  `,
})
export class LoginComponent {
  constructor(private uiBridge: UIBridgeService) {}

  async submit() {
    await this.uiBridge.executeAction('submit-btn', 'click');
  }
}

Angular Directive​

<!-- Automatic registration -->
<button uiBridge>Submit</button>

<!-- Explicit ID -->
<button uiBridge="login-btn">Login</button>

<!-- With options -->
<button [uiBridge]="{ id: 'submit', type: 'button' }">Submit</button>

Svelte Integration​

Installation​

npm install @anthropic/ui-bridge-svelte

Setup​

<!-- App.svelte -->
<script>
  import { UIBridgeProvider } from '@anthropic/ui-bridge-svelte';
</script>

<UIBridgeProvider autoRegister={true}>
  <slot />
</UIBridgeProvider>

Svelte Components​

<script>
  import { uiBridge, getUIBridge } from '@anthropic/ui-bridge-svelte';

  const { executeAction } = getUIBridge();

  async function submit() {
    await executeAction('submit-btn', 'click');
  }
</script>

<button use:uiBridge={'submit-btn'} on:click={submit}>
  Submit
</button>

Svelte Action​

<script>
  import { uiBridge } from '@anthropic/ui-bridge-svelte';
</script>

<!-- Automatic ID -->
<button use:uiBridge>Submit</button>

<!-- Explicit ID -->
<button use:uiBridge={'login-btn'}>Login</button>

<!-- With options -->
<button use:uiBridge={{ id: 'submit', aliases: ['send'] }}>Submit</button>

Vanilla JavaScript​

Installation​

npm install @anthropic/ui-bridge

Setup​

<script type="module">
  import { UIBridge } from '@anthropic/ui-bridge';

  const bridge = new UIBridge({
    autoRegister: true,
    observeDOM: true,
  });

  bridge.init();
</script>

Manual Registration​

// Register elements manually
bridge.register(document.getElementById('submit-btn'), {
  id: 'submit-btn',
  type: 'button',
  aliases: ['submit', 'send'],
});

// Or use data attributes
// <button data-ui-id="submit-btn" data-ui-type="button">Submit</button>

Execute Actions​

// Execute action by ID
await bridge.executeAction('submit-btn', 'click');

// Execute on element directly
await bridge.execute(document.getElementById('submit-btn'), 'click');

Build Tool Plugins​

Vite Plugin (Vue/Svelte)​

// vite.config.ts
import { defineConfig } from 'vite';
import uiBridge from '@anthropic/ui-bridge-vite-plugin';

export default defineConfig({
  plugins: [
    uiBridge({
      include: ['**/*.vue', '**/*.svelte'],
      idPrefix: 'ui',
    }),
  ],
});

Webpack Plugin (Angular)​

// webpack.config.js
const UIBridgePlugin = require('@anthropic/ui-bridge-webpack-plugin');

module.exports = {
  plugins: [
    new UIBridgePlugin({
      include: /\.component\.html$/,
    }),
  ],
};

Babel Plugin​

// babel.config.js
module.exports = {
  plugins: [
    ['@anthropic/ui-bridge-babel-plugin', {
      idPrefix: 'ui',
      generateAliases: true,
    }],
  ],
};

Framework Detection​

UI Bridge automatically detects frameworks:

import { detectFramework } from '@anthropic/ui-bridge';

const framework = detectFramework();
// 'react' | 'vue' | 'angular' | 'svelte' | 'vanilla'

const adapter = await loadAdapter(framework);

Consistent API​

All framework adapters provide the same core API:

interface UIBridgeAdapter {
  // Registry
  registry: UIBridgeRegistry;

  // Actions
  executeAction(id: string, action: string, value?: any): Promise<ActionResult>;
  getElementState(id: string): Promise<ElementState>;

  // Discovery
  getControlSnapshot(): Promise<ControlSnapshot>;
  findElements(query: string): Promise<Element[]>;

  // Events
  on(event: string, handler: Function): void;
  off(event: string, handler: Function): void;
}

Best Practices​

  1. Use Framework Adapters - Use the specific adapter for your framework
  2. Enable Auto-Registration - Let the plugin handle ID generation
  3. Configure Build Plugins - Set up compile-time instrumentation
  4. Test Across Frameworks - If supporting multiple, test each one
  5. Keep IDs Consistent - Use the same ID strategy across frameworks
  6. Document Framework Differences - Note any framework-specific behaviors
